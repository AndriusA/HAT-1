API Documentation

Authentication

Inbound authentication with the API is very simple. When calling an API method, just add a header called Device-ID to the HTTP request containing your unique device ID.

Outbound authentication with the API is very simple. When calling an API method, just add a header called Sensor-ID to the HTTP request containing your unique sensor ID.

Post data

To post data to the API, you must encapsulate everything as JSON in a POST field called data. An example request would be as follows

curl -H "Device-ID: x0x0x0x0-x0x0-x0x0-x0x0-x0x0x0x0x0x0"  --data 'data={"string":"A string"}' http://hat.onespace.media/api/v1/sensor_data/

The arguments required by certain methods must exist inside the JSON data that your device POSTs to the API, otherwise the API will throw an error.

POST /api/v1/sensor_data/

This method takes sensor data to be mapped to the database.

Arguments

id (string) *

Long ID of a sensor tied to the authenticating device. This argument is required.

timestamp (int)

UTC unix timestamp of the sensor reading. Not providing this argument will set it to the current server time.

value (string/array) *

The value of the sensor reading. This argument is required. If an array is supplied, this method will insert multiple entries for the same sensor.

unit (int) *

The unit type of the value as an ID. Unit IDs can be seen at the top of this documentation. This argument is required.

data_type (int) *

The data type of the value as an ID. Data type IDs can be seen at the top of this documentation. This argument is required.

description (string)

A short description about the type of data being stored.

Example request:

curl -H "Device-ID: x0x0x0x0-x0x0-x0x0-x0x0-x0x0x0x0x0x0"  --data 'data={"id":"x0x0x0x0-x0x0-x0x0-x0x0-x0x0x0x0x0x0","timestamp":1410537485,"value":10,"unit":1,"data_type":1,"description":"A value"}' http://hat.onespace.media/api/v1/sensor_data/

Example response:

{
  "response" : "success",
  "sensor_data_id" : 1
}

POST /api/v1/create_event/

This method takes data from a service such as Facebook, and maps the relevant data to the database.

Arguments

name (string) *

The name of the event. This argument is required.

description (string)

The description of the event.

type (int) *

An ID of an event type. Event type IDs can be seen at the top of this documentation. This argument is required.

status (int) *

An ID of an event status. Event status IDs can be seen at the top of this documentation. This argument is required.

time_type (int) *

An ID of an event time type. Event time type IDs can be seen at the top of this documentation. This argument is required.

start_timestamp (int) *

UTC unix timestamp of the event start time. This argument is required.

end_timestamp (int) *

UTC unix timestamp of the event end time. This argument is required.

sensor_data (array)

Array of sensor data IDs created using the sensor_data method.

locations (array)

Array of location IDs created using the create_location method.

people (array)

Array of people IDs created using the create_person method.

Example request:

curl -H "Device-ID: x0x0x0x0-x0x0-x0x0-x0x0-x0x0x0x0x0x0"  --data 'data={"name":"An event","description":"An event","type":1,"status":1,"time_type":1,"start_timestamp":1410537066,"end_timestamp":1410537866,"sensor_data":[1],"locations":[1,2],"people":[1,2,3]}' http://hat.onespace.media/api/v1/create_event/

Example response:

{
  "response" : "success"
}

POST /api/v1/create_location/

Creates an event location and returns an ID that can be used in the create_event method.

Arguments

type (int) *

An ID of a location type. Location type IDs can be seen at the top of this documentation. This argument is required.

name (string) *

The name of the location. This argument is required.

desription (string)

The description of the location.

address (object)

Address object containing the following arguments:

Arguments

house_name (string)

Address house name.

address_1 (string)

Address line 1.

address_2 (string)

Address line 2.

town (string)

Address town.

county (string)

Address county.

postal_code (string)

Address postal code.

longitude (float)

The longitude of the location.

latitude (float)

The latitude of the location.

height_from_sea_level (int)

The location's height from sea level (unknown unit).

indoor_information (string)

Indoor information?

orientation (string)

Orientation?

Example request:

curl -H "Device-ID: x0x0x0x0-x0x0-x0x0-x0x0-x0x0x0x0x0x0"  --data 'data={"name":"A location","description":"A location","type":1,"address":{"house_name":"","address_1":"123 Fake Street","address_2":"","town":"Madeupington","county":"Fakeland","postal_code":"ABC 123"},"longitude":0.138938,"latitude":52.202977,"height_from_sea_level":0,"indoor_information":"","orientation":""}' http://hat.onespace.media/api/v1/create_location/

Example response:

{
  "location_id" : 1,
  "response" : "success"
}

POST /api/v1/create_person/

Creates an event person and returns an ID that can be used in the create_event method. People are tied to the same user that owns the device sending data.

Arguments

first_name (string) *

The persons first name. This argument is required.

last_name (string) *

The persons last name. This argument is required.

Example request:

curl -H "Device-ID: x0x0x0x0-x0x0-x0x0-x0x0-x0x0x0x0x0x0"  --data 'data={"first_name":"Barry","last_name":"Scott"}' http://hat.onespace.media/api/v1/create_person/

Example response:

{
  "person_id" : 1,
  "response" : "success"
